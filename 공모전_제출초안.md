## 1) 아이디어 소개 (요약) — *300자 이내*

> 이음(以音)은 **'지역문화 데이터'를 매개로 은둔 청년과 독거노인을 잇는** AI 사회안전망이다. 매주 문화 빅데이터 플랫폼의 지역 설화·지명유래·옛 사진을 '세대공감 대화 주제'로 발행하면, 두 세대가 같은 문화 기억을 떠올리며 대화·설문에 참여한다. AI는 그 대화에서 감정·인지·성향(MBTI) 지표를 분석해 **고독사·고립 위기를 조기에 감지**하고 복지사에게 연결하며, 모인 이야기를 한 편의 수필로 엮어 세대를 잇는다. 청년의 사회 단절과 노인의 고독사라는 두 사회문제를, 문화가 가진 '공감의 힘'으로 동시에 푸는 것이 목표다. (약 290자)

---

## 2) 아이디어 상세 설명

### 2-1. 문제의식
- **노인 고독사**: 1인 가구 노인이 급증하며 사회적 관계 단절이 고독사로 이어진다. 안부 확인은 여전히 사람의 발품에 의존해 사각지대가 넓다.
- **청년 고립·은둔**: 사회와 단절된 은둔형 외톨이 청년이 늘고 있으나, 이들을 '돕는 역할'로 끌어낼 동기 설계가 부족하다.
- **세대 단절**: 두 세대는 서로의 자원이 될 수 있음에도 만날 접점이 없다.

이음은 이 셋을 **하나의 매개 — 지역문화 데이터 — 로 동시에 연결**한다. 노인에게는 익숙한 고향·옛 기억을 떠올리게 하고, 청년에게는 그 이야기를 듣고 기록하는 '의미 있는 역할'을 부여한다.

### 2-2. 핵심 사용자 흐름 (7단계 파이프라인)

```
[A] 주제 발행   문화데이터(설화·옛 사진·지명유래) → AI가 세대공감 질문 자동 생성 → 매주 발행
       │
[B] 세대 대화   노인: 음성(STT→AI→TTS) / 청년: 텍스트 + 동적 설문 응답
       │
[C] 심리 분석   대화에서 감정·어휘다양성·성향(MBTI) 등 심리지표 추출
       │
[D] 수필 생성   같은 주제의 대화들을 AI가 한 편의 수필로 엮어 세대 간 공유
       │
[E] 안전 감지   위기 키워드·접속 단절·감정 악화 감지
       │
[F] 복지사 연결 위기 시 1393·이메일 자동 알림 + 대시보드 모니터링
       │
[G] 데이터 보안 전 과정 비식별화·암호화 (횡단 적용)
```

### 2-3. 주요 기능
| 기능 | 설명 |
|------|------|
| **세대공감 주제 자동 발행** | 지역문화 데이터를 AI가 분석해 향수를 자극하는 대화 질문(선택형·서술형·혼합형)으로 변환, 매주 자동 발행 |
| **노인 음성 대화** | 말로 답하면 STT→AI→TTS로 자연스러운 음성 대화. 디지털 약자도 손쉽게 참여 |
| **청년 참여 동기 설계** | 노인의 이야기를 듣고 기록·응답하는 '기여자' 역할 부여 → 사회 재연결 유도 |
| **AI 수필화** | 흩어진 대화를 한 편의 수필로 엮어 '내 이야기가 작품이 됨'을 경험하게 함 |
| **심리·성향 분석** | 감정(긍/부/중립)·어휘 다양성(TTR)·반복표현·MBTI 성향 태깅으로 변화 추이 모니터링 |
| **위기 조기 감지·대응** | 위기 키워드/3일 이상 접속 단절/감정 악화 시 복지사에게 자동 알림, 위기 시 1393 즉시 연결 |
| **복지사 대시보드** | 담당 대상자 심리지표·위험도·개입 이력을 한눈에 관리 |

## 3) 아이디어 독창성

1. **문화데이터를 '소비 콘텐츠'가 아니라 '관계·돌봄의 촉매'로 재정의**
   기존 문화데이터 서비스는 추천·전시·관광 등 '보여주기'에 머문다. 이음은 같은 데이터를 **세대 간 대화를 여는 열쇠이자, 심리 상태를 읽는 진단 도구**로 사용한다.
2. **두 사회문제(청년 고립 + 노인 고독사)를 하나의 구조로 동시 해결**
   노인은 말벗을, 청년은 사회적 역할을 얻는 **상호 호혜 설계**. 한쪽을 돕는 행위가 다른 쪽의 치유가 된다.
3. **'향수 → 자연스러운 자기노출 → 심리 진단'의 비침습적 모니터링**
   직접적인 우울/외로움 설문 대신, 옛 기억을 떠올리는 문화 대화 속에서 감정·인지·성향을 자연스럽게 수집해 거부감을 낮춘다.
4. **대화의 수필화로 '데이터가 다시 문화가 되는' 선순환**
   문화데이터에서 출발한 대화가 새로운 창작물(수필)로 환원되어 다시 세대를 잇는다.

## 4) 아이디어 발전 가능성 및 기대효과

### 4-1. 구현·사업화 실현 가능성 — **이미 작동하는 MVP로 검증 완료**
> 본 아이디어는 책상 위 구상에 그치지 않고, **실제 동작하는 프로토타입(MVP)으로 핵심 파이프라인을 구현·검증**했다.
- **백엔드(FastAPI)**: 공공 문화데이터 API 연동, AI 주제·질문 자동 생성, 감정·위기 감지, 수필 생성, 복지사 알림(이메일/스케줄러) 동작
- **복지사 대시보드(React)**: 주제 발행·설문 관리·대상자 심리지표·위험도 모니터링 화면 구현
- **모바일 앱**: 노인 음성 대화·청년 텍스트 대화, 위기 시 1393 연결 팝업 구현

→ 따라서 사업화 시 **'개념 검증' 단계를 이미 통과**한 상태이며, 지자체·복지관 실증(파일럿)으로 곧바로 확장 가능하다.

### 4-2. 사회적 가치 (정성적 기대효과)
- **고독사 예방**: 비대면 안부·심리 모니터링으로 위기를 조기에 포착, 복지 사각지대 축소
- **청년 사회 재연결**: '누군가에게 필요한 존재'라는 역할 경험으로 은둔 탈출 동기 부여
- **세대 통합**: 문화 기억을 공유하며 세대 간 이해와 공감 회복
- **지역문화 가치 재발견**: 잊혀가던 지역 설화·지명유래가 일상 대화 속에서 다시 살아남

### 4-3. 정량적 기대효과 (지표 예시)
- 복지사 1인당 모니터링 가능 대상자 수 증가 (대시보드 자동 집계로 수기 점검 대체)
- 위기 징후 평균 감지 소요시간 단축 (정기 방문 주기 → 실시간 감지)
- 주당 세대 간 대화·수필 생성 건수 등 참여 지표
> ※ 실증 단계에서 KPI로 측정 예정 (참여율·재방문율·위기 감지 정확도·복지사 업무경감률)

### 4-4. 확장성·파급효과
- **확장**: 음원(국악)·영상 등 멀티미디어 문화데이터, 다국어(다문화가정)로 확대 가능
- **연계 산업**: 시니어 케어, 디지털 헬스케어, 지역문화 콘텐츠 산업과 결합
- **공공 파급**: 지자체 복지 행정·경로당·치매안심센터 등과 연계한 공공 돌봄 인프라화

### 활용 방식 (전처리·가공·연계)
1. **수집**: 문화 빅데이터 플랫폼에서 지역문화 데이터(CSV)를 내려받고, 국가기록원·민속아카이브 API를 실시간 연동
2. **전처리**: 이미지 URL 유효성 검증, 텍스트 설화 정제, 미디어 로컬 캐싱
3. **분할·구조화**: 이미지 유무로 `image`/`text` 주제로 분류, 핵심 키워드(TOP10) 추출 → 복지사 대시보드 필터로 제공
4. **AI 가공**: 각 데이터의 제목·설명·키워드를 AI에 입력해 **연령대별 세대공감 대화 질문**으로 변환
5. **역할**: 문화데이터는 ①대화를 여는 '주제', ②심리·성향을 읽는 '진단 매개', ③수필 창작의 '소재'라는 3중 역할을 수행한다.

> **핵심**: 문화데이터가 단순 표시 대상이 아니라, *세대를 잇고 위기를 감지하는 서비스의 작동 원리 그 자체*다.

## 6) AI 기술 활용

| 적용 영역 | AI 기술 | 역할 |
|-----------|---------|------|
| 주제·질문 생성 | 멀티프로바이더 LLM(Claude·GPT·Gemini) + 프롬프트 엔지니어링 | 문화데이터 → 연령 맞춤 세대공감 질문(선택형/서술형/혼합형) 자동 생성 |
| 세대 대화 | LLM 대화 엔진 + STT/TTS | 노인 음성↔텍스트 변환, 연령·역할별 페르소나 응답 |
| 심리 분석 | 감정분류 모델(KR-FinBert) + 언어패턴 분석(TTR·n-gram) + 성향(MBTI) 태깅 | 대화에서 감정·인지·성향 지표 추출 |
| 위기 감지 | 위기 키워드 탐지 + 규칙·패턴 기반 분류 | 자해·고립 신호를 high/medium/low로 분류해 대응 분기 |
| 수필 생성 | 생성형 LLM | 다수의 대화를 한 편의 수필로 자동 창작 |

**AI 활용 특징**
- **다중 AI 공급자 자동 전환(Fallback)**: 한 AI가 실패해도 다른 공급자로 자동 전환해 서비스 연속성 확보
- **결정적 프롬프트 + 견고한 파싱**: 동일 입력에 동일 프롬프트, 코드블록·산문이 섞인 AI 응답도 구조화 JSON으로 안정 추출
- **사람 개입(Human-in-the-loop)**: 복지사가 AI 생성 주제·질문을 검토·수정·발행하는 자동/수동 미리보기 체계로 품질·안전성 보장
