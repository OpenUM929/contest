# 14. 주석 및 로그 제거 작업 규칙

## 작업 유형
- 소스 코드 주석(`#`, `//`, `<!-- -->`, `--`, `/* */`) 일괄 제거
- 디버그 로그(`console.log`, `print`, `logger.debug` 등) 제거
- **Python docstring(`"""..."""`)은 보존**

## 핵심 원칙 (필수)
1. **제외 디렉토리 절대 준수**
   - `static/vendor/` - 외부 라이브러리 (minified 파일 손상 방지)
   - `node_modules/`, `__pycache__/`, `.git/`, `log/`, `document/`
   - 가상환경 디렉토리 (예: `venv/`, `.venv/`, `msys_venv/`)

2. **파일 패턴 제외**
   - `.min.js`, `.min.css`, `.umd.js` (압축 파일 보호)

3. **언어별 안전한 처리 방식**
   - **Python**: `tokenize` 모듈 사용 (docstring 보존, 문자열 보호)
   - **JS/HTML/SQL/CSS**: 정규식 기반 제거 (문자열 보호 로직 포함)

## 안전한 스크립트

실제 스크립트: **`remove_comments_safe.py`** (프로젝트 루트)

```bash
python remove_comments_safe.py           # 실제 실행
python remove_comments_safe.py --dry-run # 변경 예측만 출력
```

### 스크립트 핵심 아키텍처

각 언어별 처리기와 그 실행 순서가 중요:

| 언어 | 처리 함수 | 핵심 보호 장치 | 호출 순서 |
|------|-----------|---------------|-----------|
| Python | `remove_python_comments()` | `tokenize.COMMENT`만 제거 (docstring 보존) | 단일 호출 |
| JS | `remove_js_comments_and_logs()` | 문자열/템플릿 리터럴 보호 상태머신 | `/* */` → `console.*()` → `//` → `=> {}` 보정 |
| HTML | `remove_html_comments()` | `<script>`→JS처리기, `<style>`→CSS처리기 위임 | `{# #}` → script/style 처리 → `<!-- -->` |
| SQL | `remove_sql_comments_simple()` | 문자열 리터럴 `'...'` 내 `--` 보호 | `/* */` → `--` |
| CSS | `remove_css_comments_simple()` | 단순 regex | 단일 호출 |

### 언어별 금지 패턴

```python
# 🔴 절대 사용 금지 — 아래 패턴들은 코드를 파괴합니다
line[:line.find('//')].rstrip()     # http:// URL이 잘림
re.sub(r'/\*.*?
*/', '', content)   # //가 /* 내부 */ 종결자 소멸 시 파일 전체 손실
```

## 핵심 위험 패턴 (반드시 회피)

| # | 위험 패턴 | 발생 상황 | 증상 | 해결 원칙 |
|---|-----------|-----------|------|-----------|
| 1 | `//` 처리가 `/* */`보다 먼저 실행 | `/* ... // 주석 */` 에서 `//`가 `*/`를 삭제 | `/*`가 미종결 → EOF까지 블록 소비 → 함수 통째로 삭제 | **`/* */`를 항상 `//`보다 먼저 처리** |
| 2 | `//`를 단순 `line.find()`로 탐지 | `"http://url"` 문자열 내 `//` | URL 이후 모든 코드 삭제 | **문자열/템플릿 리터럴 내부의 `//`는 보호** |
| 3 | `console.*()` 제거 후 빈 화살표 함수 | `.catch(e => console.error(e))` → `.catch(e => )` | `Uncaught SyntaxError: Unexpected token ')'` | **`re.sub(r'(=>)
\s*\)', ...)` post-processing** |
| 4 | `/*` 미종결 상태 내버려둠 | 사례 1의 결과 또는 원천 코드 결함 | `/*`가 EOF까지 소비 → 다수 함수 삭제 | **block comment 미종결 시 원본 보존** |
| 5 | `console.*()` 괄호 단순 regex 매칭 | `console.log(foo(sth))` 에서 `[^)]*` 오작동 | `)`가 문자열 내에 있을 때 매칭 실패 | **paren-counting 상태머신 사용** |
| 6 | HTML `<script>` 블록 무시 | `.html` 파일 내 inline JS의 `//` 주석 | 주석이 제거되지 않고 남음 | **`<script>` 블록 추출 → JS 처리기 위임** |
| 7 | Jinja2 `{# #}` 미처리 | `{# 주석 #}` 은 HTML 표준 주석이 아님 | 제거되지 않고 렌더링되어 클라이언트 노출 | **별도 regex `
\{#.*?#\}` 필요** |

## 장애 사례 및 복구

### 사례 1: 가상환경 패키지 손상
- **증상**: `re.error: bad character range`
- **원인**: 가상환경 디렉토리 내부 파일 수정
- **해결**: 가상환경 재생성 + 의존성 재설치
  ```bash
  cd ium
  python -m venv venv
  .\venv\Scripts\pip install -r requirements.txt
  ```

### 사례 2: Vendor JS 파일 손상
- **증상**: `SyntaxError: Invalid or unexpected token`
- **원인**: `.min.js` 파일에 주석 제거 정규식 적용
- **해결**: 원본 복구
  ```bash
  git checkout -- static/vendor/
  ```

### 사례 3: `console.*()` 제거 후 빈 화살표 함수
- **증상**: `Uncaught SyntaxError: Unexpected token ')'`
- **원인**: `.catch(error => console.error('...'));` → console.*()만 제거되어 `.catch(error => );` 잔여
- **해결**: `remove_js_comments_and_logs()` 내 `re.sub(r'(=>)
\s*\)', r'\1 {})', content)` post-processing 적용
- **검증**: `grep -rn '=>
\s*)' --include='*.js' --include='*.html'`로 잔여 패턴 확인

### 사례 4: `/* */` 블록 내부 `//`가 `*/` 종결자 소멸
- **증상**: 함수 전체가 삭제되어 `Unexpected token 'export'` / `Unexpected token 'class'`
- **원인**:
  ```javascript
  /* ...설명...
     some code; // 주석 */    ← `//` 제거가 `*/`까지 삭제
  ```
  `*/`가 사라지면서 `/*`가 미종결 상태가 되어 파일 끝까지 블록으로 인식
- **해결**: `remove_js_comments_and_logs()`는 **반드시 `/* */` → `//` 순서**로 처리
- **복구**: `backup_202*`에서 원본 복원 후 재처리
- **검증**: `node --check <file>`로 각 JS 파일 문법 검증

### 사례 5: URL `http://` 내 `//`를 주석으로 오인
- **증상**: `Uncaught SyntaxError: Invalid or unexpected token`
- **원인**: `line.find('//')`가 `"http://..."` 의 `//`를 주석으로 착각, URL 이후 전부 삭제
- **해결**: `_process_js_line()` 문자열 리터럴/템플릿 리터럴 인식 로직으로 보호
- **복구**: `backup_202*`에서 원본 복원 후 재처리
- **검증**: `node --check <file>`로 각 JS 파일 문법 검증
