# 이음(以音) — 간단 사용 가이드

> **한 줄 요약**: 문화 공공데이터를 주제로 노인과 청년이 AI와 대화하고, 대화를 모아 익명 수필을 만드는 사회안전망 플랫폼

## 2. 사전 준비

### 필요한 도구
- **Docker Desktop** (백엔드 + DB 실행용)
- **Node.js** (프론트엔드 실행용)
- **API 키** 3개
  - `ANTHROPIC_API_KEY` — AI 대화 (Claude)
  - `OPENAI_API_KEY` — 음성 변환 (Whisper) + TTS
  - `PUBLIC_DATA_API_KEY` — 공공데이터 (선택)

### 환경 설정 (1분)

```bash
cd C:\dev\contest\ium
cp .env.example .env
# .env 파일을 메모장으로 열어 API 키 입력
```

`.env` 예시:
```
ANTHROPIC_API_KEY=sk-ant-...
OPENAI_API_KEY=sk-...
PUBLIC_DATA_API_KEY=your_key_here
```

> 💡 Clova Speech/Voice는 `.env`에 `CLOVA_CLIENT_ID`, `CLOVA_CLIENT_SECRET`을 넣으면 자동으로 전환됩니다.

## 3. 실행 방법

### 3-1. 백엔드 + DB 시작 (Docker)

```bash
cd C:\dev\contest\ium
docker-compose up -d
```

실행 확인:
```bash
curl http://localhost:8000/health
# {"status":"ok","service":"이음(以音)"}
```

> 띄우는 순서: PostgreSQL → Redis → FastAPI (자동으로 순서대로 시작됨)

### 3-2. 복지사 대시보드 시작 (웹)

```bash
cd C:\dev\contest\ium\dashboard
npm install
npm run dev
# http://localhost:5173
```

### 3-3. 모바일 앱 시작 (Expo)

```bash
cd C:\dev\contest\ium\mobile
npm install
npx expo start
# QR 코드를 Expo Go 앱으로 스캔
```

> ⚠️ `ElderlyMain.tsx`에 `http://localhost:8000`이 하드코딩되어 있어, 실제 기기에서는 PC IP 주소로 바꿔야 합니다.

## 4. 주요 기능 사용법

### 4-1. 사용자 등록

```bash
curl -X POST http://localhost:8000/chat/users \
  -H "Content-Type: application/json" \
  -d '{"user_type": "elder", "nickname": "홍길동 어르신"}'
```

> `user_type`: `"elder"` (노인) 또는 `"youth"` (청년)

### 4-2. 이번 주 주제 확인

```bash
curl http://localhost:8000/chat/topic
```

### 4-3. 대화하기

```bash
curl -X POST http://localhost:8000/chat/message \
  -H "Content-Type: application/json" \
  -d '{"user_id": "위에서 받은 UUID", "message": "남대문 시장 생각나네요"}'
```

AI 반응:
- **노인**: 존댓말, 짧은 문장, 공감 후 질문
- **청년**: 편한 말투, 판단 없음, 익명 보장 언급
- **위기 키워드** 감지 시 자동으로 1393 안내 문구 삽입

### 4-4. 수필 생성 (5건 이상 대화 후)

```bash
curl -X POST http://localhost:8000/essay/generate
```

> 같은 주제로 5건 이상 대화가 쌓여야 생성됩니다.

### 4-5. 복지사 대시보드 데이터 확인

```bash
curl http://localhost:8000/welfare/dashboard
```

반환 예시:
```json
{
  "red": ["위험 사용자 목록"],
  "yellow": ["주의 사용자 목록"],
  "green": ["정상 사용자 목록"]
}
```

### 4-6. 안전 점검

```bash
curl http://localhost:8000/safety/check/{user_id}
```

## 6. 주요 기능 요약

| 기능 | 설명 | 파일 |
|---|---|---|
| **AI 대화** | 주제 기반 대화, 노인/청년 말투 자동 분기 | `services/claude.py` |
| **수필 생성** | 5건 이상 대화 모아 익명 수필 자동 작성 | `services/essay.py` |
| **감정 분석** | 대화 감정을 긍정/부정/중립으로 분류 | `services/emotion.py` |
| **안전 감지** | 3일 미접속, 위기 키워드(고립·자해 의도) 감지 | `services/alert.py` |
| **음성 인터페이스** | 음성 입력 → Whisper STT → AI 응답 → TTS 재생 | `services/voice.py` |
| **복지사 알림** | 위험도 🔴🟡✅ 분류 및 대시보드 제공 | `routers/welfare.py` |

## 8. 자주 하는 실수

| 문제 | 원인 | 해결 |
|---|---|---|
| `docker-compose up` 실패 | Docker Desktop 미실행 | Docker Desktop 먼저 켜기 |
| API 응답 없음 | `.env`에 API 키 미입력 | `.env` 파일 확인 |
| 모바일 앱이 백엔드 못 찾음 | `localhost` 하드코딩 | PC의 실제 IP 주소로 변경 |
| 수필 생성 안 됨 | 대화 5건 미만 | 더 대화한 후 재시도 |
