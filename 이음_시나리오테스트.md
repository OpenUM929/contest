## SC-01 | 주제 선정 파이프라인

### 개요

| 항목 | 내용 |
|---|---|
| 목적 | 공공데이터 유형(사진/음악/이야기)별로 주제가 올바르게 선정되고 앱에 전달되는지 검증 |
| 관련 코드 | `services/public_data.py` — `TOPIC_POOL`, `get_active_topic()` |
| API | `GET /chat/topic` |

---

### 파이프라인 흐름 — 공통

```
[매주 월요일 00:00]
    │
    ▼ Step 1. 현재 주차(ISO Week) 계산
    │  today = date.today()
    │  monday = today - timedelta(days=today.weekday())
    │  week_number = monday.isocalendar().week  → 예: 23
    │
    ▼ Step 2. 주제 풀 인덱스 결정
    │  idx = week_number % len(TOPIC_POOL)  → 23 % 5 = 3
    │  → TOPIC_POOL[3] 선택 = "1980년대 동네 골목 풍경"
    │
    ▼ Step 3. 미디어 타입 분기
    │  media_type == "image"  → 사진 경로
    │  media_type == "audio"  → 음원 URL
    │  media_type == "story"  → 텍스트 본문 (향후)
    │
    ▼ Step 4. AI 질문 반환
       ai_question: "어릴 때 살던 동네 골목은 어떤 모습이었나요?"
```

---

### 케이스 1-A: 사진(image) 주제 선정

**전제 조건**: 현재 주차 idx → `TOPIC_POOL[0]` 또는 `TOPIC_POOL[3]` 선택

**입력**:
```bash
curl http://localhost:8000/chat/topic
```

**파이프라인**:
```
Step 1. get_active_topic() 호출
Step 2. media_type = "image" 확인
Step 3. 앱 렌더링 분기
    │  노인 앱: [사진 카드] + TTS("이번 주 이야기입니다. 1960년대 서울...")
    │  청년 앱: [사진 카드] + 텍스트 입력창
Step 4. ai_question 반환 → 앱 화면 하단에 표시
```

**예상 응답**:
```json
{
  "title": "1960년대 서울 남대문 시장",
  "description": "분주한 장터의 활기 속에 사람들의 일상이 담겨 있습니다.",
  "media_type": "image",
  "media_url": null,
  "source": "국가기록원",
  "source_url": "https://www.archives.go.kr",
  "ai_question": "이런 시장에 가보신 적 있으신가요? 그때 어떤 것을 사셨나요?",
  "active_week": "2026-06-01"
}
```

**앱 화면 반응**:
```
[노인 앱]
┌────────────────────────────────┐
│  이번 주 이야기                 │
│  ┌──────────────────────────┐  │
│  │  📷 1960년대 남대문 시장  │  │  ← 사진 카드
│  │  [국가기록원 제공]        │  │
│  └──────────────────────────┘  │
│  🔊 "이런 시장에 가보신 적      │  ← TTS 자동 낭독
│      있으신가요?"               │
│  ┌──────────────────────────┐  │
│  │  🎤  말씀해 주세요        │  │
│  └──────────────────────────┘  │
└────────────────────────────────┘

[청년 앱]
┌────────────────────────────────┐
│  이번 주 주제                   │
│  📷 1960년대 남대문 시장        │  ← 사진 카드 (재생 버튼 없음)
│  [국가기록원 제공]              │
│  "이런 시장에 가보신 적         │
│   있으신가요?"                  │
│  ┌──────────────────────────┐  │
│  │ 자유롭게 써주세요         │  │  ← 텍스트 입력
│  └──────────────────────────┘  │
└────────────────────────────────┘
```

**실패 케이스**:
```
- TOPIC_POOL 비어있음 → IndexError → HTTP 500
  조치: TOPIC_POOL 최소 1건 이상 보장
- media_url = null (API 미연동 상태)
  조치: 앱에서 null 체크 후 placeholder 이미지 표시
```

---


### 파이프라인 흐름

```
[Step 1] 노인 사용자 등록
    │  POST /chat/users
    │  { "user_type": "elder", "nickname": "이순자 어르신",
    │    "welfare_id": "<welfare_uuid>" }
    │  → user_id 반환
    │
    ▼
[Step 2] 주제 조회 및 TTS 낭독
    │  GET /chat/topic
    │  → topic 객체 반환
    │  앱: Speech.speak("이번 주 이야기입니다. " + topic.title
    │                  + ". " + topic.ai_question)
    │  속도: rate=0.85 (노인용 느린 속도)
    │
    ▼
[Step 3] 노인 음성 입력 → STT
    │  노인이 🎤 버튼 누른 채 말함
    │  Audio.Recording 시작 → 버튼 떼면 자동 종료
    │  POST /chat/voice/stt (audio.m4a 업로드)
    │  → Whisper API 호출
    │  → { "text": "남대문 시장 생각나네요. 어머니 손잡고..." }
    │
    ▼
[Step 4] AI 대화 요청 (스트리밍)
    │  POST /chat/message
    │  { "user_id": "<uuid>", "message": "남대문 시장 생각나네요..." }
    │
    ├── [4-1] 사용자 조회 → user.user_type = "elder" 확인
    ├── [4-2] last_seen_at = NOW() 갱신
    ├── [4-3] get_active_topic() → 이번 주 주제 로드
    ├── [4-4] 최근 대화 10건 로드 (컨텍스트)
    ├── [4-5] Conversation(role="user") DB 저장
    ├── [4-6] detect_crisis_level("남대문 시장...") → None (정상)
    ├── [4-7] emotion.analyze("남대문 시장 생각나네요...")
    │         → { "label": "positive", "score": 0.78 }
    │         → user_conv.emotion_label = "positive" 저장
    └── [4-8] claude.stream_chat(user_type="elder", ...) 호출
              ELDER_SYSTEM 프롬프트 적용
              → 스트리밍 응답 반환
              → 앱: 텍스트 표시 + TTS 자동 낭독
    │
    ▼
[Step 5] AI 응답 저장
    │  Conversation(role="assistant", content="어머니랑 함께...") 저장
    │
    ▼
[Step 6] 대화 반복 (Step 3~5 반복, 주당 평균 3~5회)
    │
    ▼
[Step 7] 수필 수신 (주말 배포)
       GET /essay/latest
       → { "title": "기억 속의 장터",
           "content": "어떤 사람에겐...",
           "contributor_cnt": 23 }
       앱: 수필 화면 표시 + TTS 낭독 옵션
```

## SC-03 | 청년 사용자 전체 흐름 (텍스트 익명 기반)

### 개요

| 항목 | 내용 |
|---|---|
| 목적 | 청년 사용자가 익명으로 텍스트 대화하고 수필을 수신하는 전체 흐름 검증 |
| 사용자 | youth 타입, 익명, 텍스트 중심, 다크모드 |
| 관련 코드 | `routers/chat.py`, `services/claude.py`, `services/emotion.py` |

---

### 파이프라인 흐름

```
[Step 1] 청년 익명 가입
    │  POST /chat/users
    │  { "user_type": "youth" }
    │  → nickname: "익명_a3f7c2" (자동 생성)
    │  → is_anonymous: true
    │
    ▼
[Step 2] 주제 조회
    │  GET /chat/topic
    │  media_type = "audio" 라면 → 음악 재생 버튼 표시
    │  media_type = "image" 라면 → 사진 카드 표시
    │  ai_question → 입력창 위에 표시
    │
    ▼
[Step 3] 텍스트 입력 → 전달하기
    │  사용자가 입력창에 작성
    │  "전달하기" 버튼 → POST /chat/message
    │  { "user_id": "<youth_uuid>",
    │    "message": "이 노래 들으니까 할머니 생각나요. 항상 이런 노래..." }
    │
    ▼
[Step 4] 서버 처리 파이프라인
    ├── [4-1] user_type = "youth" → YOUTH_SYSTEM 프롬프트 적용
    ├── [4-2] last_seen_at 갱신
    ├── [4-3] detect_crisis_level() → None (정상)
    ├── [4-4] emotion.analyze("이 노래 들으니까 할머니 생각나요...")
    │         → { "label": "neutral", "score": 0.61 }
    └── [4-5] claude.stream_chat(user_type="youth", ...)
              YOUTH_SYSTEM 프롬프트: 편안한 말투, 익명 보장 상기
              → "할머니 생각이 나셨군요. 어떤 노래를 즐겨 부르셨어요?"
    │
    ▼
[Step 5] 응답 표시 (스트리밍, 타이핑 효과)
    │  aiResponse 상태 업데이트 → 실시간 텍스트 렌더링
    │
    ▼
[Step 6] 대화 반복
    │
    ▼
[Step 7] 수필 수신 확인
       GET /essay/latest
       앱: "📖 이번 주 수필 보기" 탭 → 수필 표시
       → "👆 나도 기여했어요 (19번째)" 표시
```

## SC-07 | 감정 악화 추세 감지

### 개요

| 항목 | 내용 |
|---|---|
| 목적 | 7일간 감정 지표 추이를 분석해 위험 수준을 자동 분류하는 흐름 검증 |
| 트리거 | 최근 7일 대화 감정 레이블 기반 |
| 관련 코드 | `services/emotion.py` — `check_emotion_trend()`, `services/alert.py` |

---

### 감정 추세 분류 기준

```python
# check_emotion_trend() 로직
최근 7일 감정 레이블 목록 → recent

len(recent) < 3   → None (데이터 부족)
last_three_all_neg = all(e["label"] == "negative" for e in recent[-3:])
neg_count = sum(1 for e in recent if e["label"] == "negative")

last_three_all_neg AND neg_count >= 5  → "red"
neg_count >= 5                          → "yellow"
그 외                                   → None
```

---

### 파이프라인 흐름

```

    │  INSERT INTO conversations (user_id, role, content,
    │      emotion_label, emotion_score, created_at)
    │  VALUES (uuid, 'user', '오늘도 힘들었어요',
    │          'negative', 0.82, NOW() - INTERVAL 'N days')
    │  N = 6, 5, 4, 3, 2, 1, 0
    │
    ▼
[Step 2] 안전 점검 API 호출
    │  GET /safety/check/<user_id>
    │
    ▼
[Step 3] get_recent_emotions() 실행
    │  SELECT emotion_label, created_at
    │  FROM conversations
    │  WHERE user_id = <uuid>
    │    AND role = 'user'
    │    AND emotion_label IS NOT NULL
    │    AND created_at >= NOW() - 7일
    │  → recent = [
    │       {"label": "negative", "date": "2026-05-26"},
    │       {"label": "negative", "date": "2026-05-27"},
    │       {"label": "negative", "date": "2026-05-28"},
    │       {"label": "negative", "date": "2026-05-29"},
    │       {"label": "negative", "date": "2026-05-30"},
    │       {"label": "negative", "date": "2026-05-31"},
    │       {"label": "negative", "date": "2026-06-01"},
    │    ]
    │
    ▼
[Step 4] check_emotion_trend() 평가
    │  neg_count = 7 ≥ 5  → 조건 충족

    │  → return "red"
    │
    ▼
[Step 5] 알림 생성
    │  trigger_alert(db, user_id, "emotion_drop", "red",

    │
    ▼
[Step 6] 대시보드 반영
       GET /welfare/dashboard
       → user.active_alerts = 1
       → user.latest_emotion = "negative"
       → level = "red"
       → [🔗 상담 연결] 버튼 활성화
```
