## 1. Executive Summary

**목표:** 리더십 텍스트에서 마이크로 라벨을 추출하고 Trait을 추론하는 시스템 구축  
**핵심 발견:** Vector DB 단독으로는 부정 문장 처리(T105-A)와 맥락 판단(T04-B)에서 한계가 뚜렷함  
**현재 상태:** 178개 라벨, KoE5 + FAISS + LLM 하이브리드 파이프라인 운영 중

---

## 2. 시스템 아키텍처 비교

### 2.1 Vector DB 단독 (폐기)

```
[입력 텍스트] → [임베딩] → [Vector 검색] → [라벨 추천] → [Trait 매칭] → [결과]
```

| 구성요소 | 구현 방식 | 한계 |
|-----------|----------|------|
| 임베딩 | KoE5 (한국어) | 정적 — 추론 능력 없음 |
| Vector DB | FAISS (로컬) | 메타데이터 필터링 제한적 |
| 맥락 판단 | ❌ 없음 | T04-B 실패 원인 |
| 부정 처리 | ❌ 불가 | T105-A 실패 원인 |

### 2.2 Vector DB + LLM 하이브리드 (현재 운영)

```
[입력 텍스트]
    ↓
[Vector DB] → [상위 15개 후보 라벨] (빠른 사전 필터링)
    ↓
[LLM] → [맥락 할당 + 부정 검출 + 신뢰도] (추론 레이어)
    ↓
[Trait 엔진] → [최종 결과]
```

| 구성요소 | 역할 | 가치 |
|-----------|------|-----|
| Vector DB | 검색 공간 축소 | LLM 호출 비용 절감 |
| LLM | 맥락 판단, 부정 검출, 신뢰도 | T04-B, T105-A 해결 |
| Trait 엔진 | required/optional 조건 강제 | 정확한 Trait 추론 |

## 3. 실패 사례 분석 (해결됨)

### 3.1 T04-B (위기 관리자)

**입력:** "문제가 발생했다. 지금 상황을 분석하고 있다. 빠르게 결론을 내릴 것이다."

| 접근법 | M19-01 | M20-01 | 맥락(crisis) | 결과 |
|--------|--------|--------|--------------|------|
| Vector 단독 | ✅ | ✅ | ❌ normal | **실패** |
| Vector + LLM | ✅ | ✅ | ✅ crisis | **성공** |

### 3.2 T105-A (조작적 리더십)

**입력:** "이번 quarter 성과 좋으면 즉시 승진시켜드립니다. 하지만 실제로는 그런 일 없습니다."

| 접근법 | N02-02 | N02-03 | 전체 의도 분석 | 결과 |
|--------|--------|--------|----------------|------|
| Vector 단독 | ⚠️ | ⚠️ | ❌ 별개 매치 | **실패** |
| Vector + LLM | ✅ | ✅ | ✅ 연속성 이해 | **성공** |

## 5. 현재 성능

| 지표 | Vector 단독 | Vector + LLM |
|------|-------------|--------------|
| 부정 처리 | 20-30% | 85-90% |
| 맥락 판단 | 40-50% | 80-90% |
| 전체 정확도 | 50-60% | **85-95%** |
| 응답 속도 | 빠름 | 중간 (LLM 호출 시) |
| 비용 | 무료 | 무료 (OpenCode) |
