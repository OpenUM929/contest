# 경량 AI(KOTE식) 도입 타당성 연구 로그

> 작성일: 2026-06-20
> 목적: 미구현 상태인 AI 모듈을 KOTE식 경량 분류기로 구현 가능한지 검증
> 상태: **PoC 1차 완료** — baseline 측정 및 핵심 인사이트 도출, hard eval은 미착수

---

## 0. 한 줄 요약

> KOTE식 경량 분류기는 **도입 가치가 있다(floor 69% > 현 하이브리드 추정 55%)**.
> 그러나 우리가 세운 **"분류기로 강점 / 추론으로 리스크" 가설은 현 데이터로 검증 불가**하며,
> 측정 결과는 오히려 **데이터셋이 hard 케이스를 담지 못한 한계**를 드러냈다.
> 진짜 검증의 관문은 **hard eval set(맥락 뒤집힘·문장 간 의도)** 구축이고, 여기서 다면평가 실데이터가 결정적이다.

---

## 1. 배경 & 목표

- 현 프로젝트는 RAG·AI 모듈이 **미구현**. AI는 KOTE(한국어 감정 50k·43라벨, KcELECTRA fine-tune) 방식으로 만들고자 함.
- 발상: **회사 다면평가 결과 → trait·micro label 매핑·학습 → 경량 분류기**. 궁극적으로 RAG + 경량 AI로 문서상의 리더십을 판단.
- 본 연구는 "그게 가능한가"를 **수치로** 확인하는 것이 목표.

### 현 시스템 자산 (확인됨)
- 178개 마이크로 라벨(긍정 M 98 + 부정 N 80) → 18개 trait(T01~T14 긍정 / T101~T106 부정)
- 학습 데이터 `dataset/ori/training_data_all_labels.json`: **12,460 문장**, 라벨당 70개, `clean`/`hard_negative`/blank 구분
- 현 파이프라인: KoE5 임베딩 + FAISS + LLM 하이브리드(분류기 학습은 미사용)

## 2. 가설 (예상했던 것)

| # | 가설 | 근거 |
|---|------|------|
| H1 | KOTE식 경량 분류기 학습은 가능하다 | 라벨링 데이터가 이미 KOTE식 포맷으로 존재 |
| H2 | trait·micro 라벨은 KOTE 감정과 **결이 다르다** | 감정=글쓴이 정서(표면), 리더십=제3자 행동 판단(화용 추론) |
| H3 | **부정 라벨이 분류기에 어렵다** | 역접·부정·맥락 뒤집힘. vector-only가 부정에서 20~30%로 실패했던 이력 |
| H4 | "분류기로 강점(긍정) / 추론으로 리스크(부정)" 이분법이 성립 | H3의 연장. 설계자도 부정 trait을 `hybrid_eligible:false`로 둠 |

## 3. 방법

### 3-A. 구조신호 기반 A/B 라벨 분류 (`scripts/classify_labels_ab.py`)
178개 라벨 각각에 **추론 난이도 신호**를 점수화:

| 신호 | 점수 |
|------|------|
| 부정 라벨(역접·부정 처리) | +2 |
| 맥락 promotion/demotion (`context_rules.json`) | +2 |
| 긍/부정 미러(M19↔N19형, 맥락절로 뒤집힘) | +2 |
| 은닉 의도/화용(조작·은폐·가로채기 등) | +2 |
| trait hard_forbidden / soft_forbidden | +1 / +1 |
| 맥락 가중치 편차 ≥0.4 | +1 |

→ A(0~1) / C(2~3) / B(≥4). 결과: `docs/label_ab_classification.md`

### 3-B. Baseline 분류기 PoC (`scripts/baseline_classifier_poc.py`)
- 벡터화: char n-gram(2~5) **TF-IDF** — 의미임베딩 없는 **보수적 하한(floor)**
- 모델: LogisticRegression(`class_weight=balanced`)
- 학습 데이터 처리: **clean+blank = 양성 학습**, **hard_negative = 거부(rejection) 테스트로 분리**
  (hard_negative는 "그 라벨의 반례"이므로 양성으로 학습하면 오염)
- 라벨별 75/25 분할, 라벨별 F1을 A/B/C 버킷과 교차표. 결과: `docs/baseline_classifier_report.md`

### 3-C. 버킷 v2 보정 (`scripts/merge_buckets_v2.py`)
구조신호 + 경험적 F1 결합. 구조적 추론위험(B)은 유지, 그 외는 F1로 재배치.

> 환경: scikit-learn 1.9.0 설치(PoC용). torch/sentence-transformers 미설치라 KcELECTRA·KoE5 미사용.

## 4. 결과 (수치)

### 4-1. 구조신호 A/B 분류 (3-A)
| 버킷 | 의미 | 전체 | 긍정 | 부정 |
|------|------|------|------|------|
| A | 분류기 적합 | 16 | 16 | 0 |
| C | 분류기+규칙 | 73 | 73 | 0 |
| B | 추론 전담 | 89 | 9 | **80** |

- 부정 80개 **전부** B. 자동 분류 경계가 설계자의 `hybrid_eligible` 경계와 일치 → 가설 H3·H4를 지지하는 듯 보였음.

### 4-2. Baseline PoC (3-B) — **여기서 반전**
- **전체 Top-1 = 0.692, macro-F1 = 0.689** (TF-IDF floor인데 현 하이브리드 추정 55% 상회)

| 버킷(구조) | 예측 | **실측 평균 F1** |
|---|---|---|
| A "쉬움" | 높을 것 | **0.551 (최하)** |
| C | 중간 | 0.640 |
| B "어려움" | 낮을 것 | **0.746 (최상)** |
| **긍정 전체** | — | 0.630 |
| **부정 전체** | — | **0.757** |

- hard_negative 거부: 오인율 0.380, **평균 확신도 0.177**(낮음 → 게이트로 거부 가능)
- 173/178 라벨만 평가됨(5개는 양성 샘플 없음 = 전량 hard_negative)

### 4-3. 버킷 v2 (3-C)
| | v1(구조) | **v2(보정)** |
|---|---|---|
| A 분류기 단독 신뢰 | 16 | **34** (F1 0.76~0.89) |
| C+B* 규칙/계층 보정 | 73 | 50 |
| B 추론 전담 | 89 | 89 |

- A에서 **orphan 거품 9개 강등**(M44-02 0.27, M46-02 0.32, M29-01 0.40, M36계열 등)
- 신규 **B\*(16개)**: 추론위험 낮은데 서로 혼동 → **라벨 병합/계층화 후보**
  (M15 심리적안전 6개 0.33~0.42, M28-02 정보공유 0.20 등)

## 5. 핵심 인사이트

### I1. 분류기 도입은 정당하다
의미임베딩 없는 최하한도 69%. KcELECTRA·KoE5는 이 위. LLM 호출을 대폭 줄일 1차 필터로 충분.

### I2. **가설 H3·H4는 현 데이터로 검증 불가 (측정이 가설을 반증한 게 아니라, 측정이 불가능했음)**
- 구조신호상 "어려운" 부정 라벨이 실측에선 **제일 잘** 맞았다(0.757).
- 원인: **데이터셋이 한 문장=한 라벨의 고립·clean 샘플**. 부정은 고립되면 어휘가 뚜렷(해고·협박·가로채기·조작)해 TF-IDF가 쉽게 잡는다.
- **정작 어려운 케이스(맥락 뒤집힘 M19↔N19, "하지만 실제로는", 문장 간 의도)는 데이터에 없다.**
- 따라서 부정 F1 0.757은 "부정이 쉽다"가 아니라 **"테스트셋이 hard 케이스를 안 담았다"는 artifact**.
- 교훈: **구조신호 휴리스틱은 버킷 기준으로 부적합**. 경험적 F1로 갈아타야 함(= v2의 동기).

### I3. A버킷이 낮았던 진짜 이유 = orphan 추상 라벨
"쉽다"던 A에 trait에도 안 묶인 추상 개념(성취지향·기회인식·M36계열)이 들어가 다른 긍정 라벨과 겹침. **구조신호 0 = 쉬움이 아니라 = trait 미연결**이었음.

### I4. B\*는 추론 문제가 아니라 **라벨 설계 문제**
M15(심리적안전) 6개, M28(정보공유)처럼 한 macro를 과도하게 잘게 쪼갠 라벨들이 서로 혼동. → 추론으로 풀 게 아니라 **병합 또는 2단(macro→micro) 분류**로 풀 문제.

### I5. 2단계 라우팅 설계가 데이터로 뒷받침됨
hard_negative 평균 확신도 0.177 → 확신도 게이트(~0.35)로 반례 대부분이 LLM 경로로 빠진다. "분류기 확신↑→채택 / 확신↓→추론" 구조 타당.

## 7. 다음 단계 (우선순위)

1. **[최우선] Hard eval set 구축** — RAG_PLAN flip 케이스(M19↔N19), 역접("하지만 실제로는"), 문장 간 의도 등 맥락 의존 케이스 수십~수백 개. 여기서 **분류기 vs LLM 격차**를 처음으로 진짜 측정. → **다면평가 실데이터가 결정적**(실제 코멘트엔 복합 문장이 많음).
2. **의미임베딩 baseline** — torch+KcELECTRA/KoE5로 floor 대비 상한 확인.
3. **B\* 라벨 재설계** — M15·M28 계열 macro 병합 또는 2단 분류.
4. **다면평가 결합 설계** — 점수→trait 매핑은 가능, 코멘트→micro는 기존 파이프라인으로 weak-label 후 검수. PII 비식별화·긍정편향·문체갭 유의.
