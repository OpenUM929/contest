# 출처·학술근거 강화 방법 (정당성 상승: ~4.2 → 4.6+)

> 2026-06-20 · 사용자: 출처·학술근거가 우리 taxonomy의 *정당성*을 얻는 핵심 데이터.
> 목적: 현재 ~4.2에서 막힌 두 차원을 **데이터 없이 조사로** 4.6+까지 올리는 구체적 방법.

## 0. 왜 지금 ~4.2에서 막혔나 (정직한 진단)
| 차원 | 현재 한 일 | 막힌 이유(상한 원인) |
|---|---|---|
| 출처 품질 | DOI·peer_reviewed·등급(M/S/E/R/B)·claim_quote | ① **단일 출처** 구성개념 다수(삼각검증 부재) ② claim_quote가 *내 요약*(페이지/원문 미검증) ③ 강도 정보 없음(표본수·효과크기·이질성) ④ 일부 **오래된 단일 근거**(2002/2008) ⑤ Schmidt=**thesis**(비동료심사) |
| 학술 근거 | trait마다 차원 부모(dimensional_parent)·메타 인용 | ⑥ 매핑이 **'차원 라벨' 수준**(검증된 측정 *문항*과 대응 안 됨) ⑦ **단일 프레임워크** 의존(수렴 근거 부족) ⑧ **단일 평정자(나)** 매핑(코딩 프로토콜·재현성 없음) ⑨ 노몰로지컬 네트워크(기대 상관) 미기록 |

## 1. 출처 품질 → 4.6 (방법)

### M1. GRADE-lite 강도 메타데이터 (가장 큰 rigor 점프)
출처에 정량 강도 필드 추가: `n_studies`(메타 k) / `N`(표본) / `effect_size`(+CI) / `heterogeneity` / `journal_tier`(SJR Q1~Q4) / `replication`(독립 복제 수).
→ "peer_reviewed=true" 이분값을 **정량 신뢰도**로 격상. 예: 서번트 +12%가 "k=39, N=3,000+, Q1"이면 주장의 무게가 객관화.

### M2. 독립 출처 삼각검증 (단일 출처 제거)
각 핵심 정량 주장에 **독립 출처 ≥2**. 단일 출처 구성개념을 `source_rigor_audit.py`로 플래그 → 보강 검색.
→ 단일 인용 = 재현성 약함. 독립 2개 = 수렴 증거.

### M3. 최신성 게이트
구성개념마다 **최근 5년(≥2021) 출처 ≥1** 또는 "안정된 고전(업데이트 불요)" 명시. 오래된 단일 근거(예: laissez-faire 1994)는 최신 메타로 보강.

### M4. 검증가능 인용(verbatim + locator)
`claim_quote`를 **페이지/표 + 원문 표현**으로. FACT_CHECK를 사람이 대조 → `verified:true/날짜`. (사용자 협업)

### M5. 학술지 품질 스크린
`journal_tier`(SJR/JCR 분위) 기록 → 약탈적·저품질 저널 배제. Q1/Q2 비율을 지표화.

### M6. thesis/2차 보강
Schmidt(2008 thesis) → 동료심사 TLS **검증/복제 논문**으로 보강 인용(독립 corroboration 겸).

## 2. 학술 근거 연결 → 4.6 (방법)

### M7. 척도-문항 대응 (가장 강력한 정당성 근거) ⭐
각 trait의 micro를 부모 구성개념의 **검증된 측정도구 문항**에 대응시킨다:
- T15 Empowering ↔ **Ahearne/Spreitzer** empowering/심리적임파워먼트 척도 문항
- T07 Coaching/Servant ↔ **Liden SL-7** developing 문항
- T09 Visionary ↔ **MLQ** Inspirational Motivation/Idealized Influence 문항
- T102 Authoritarian ↔ **Schmidt TLS** authoritarian 문항
→ "차원 라벨" → "검증된 문항의 조작화"로 격상 = 측정학적 정당성. (`scale_item_map.json`)

### M8. 수렴 삼각검증(이론×실증)
각 부모를 **framework + meta(또는 독립 실증) ≥2**로. 단일 프레임워크 의존 제거.

### M9. 노몰로지컬 네트워크
각 구성개념의 **기대 outcome 상관**(메타값: 직무만족·성과·이직 등) 기록 → 준거타당도 근거 + *향후 다면평가 데이터의 대조점*(예측을 미리 박아두면 실증 단계에서 검증력↑).

### M10. 코딩 프로토콜(코드북) + 2차 독립 코딩
구성개념→trait 매핑 규칙을 **코드북**으로 문서화(재현성) + **2차 독립 코딩**(외부 전문가 또는 별도 에이전트)로 단일 평정자 편향 제거. 일치도(κ) 기록.

## 3. 우선순위 (정당성 레버 순)
1. **M7 척도-문항 대응** — "검증된 도구에 정초" = 최강 정당성. (학술 4.2→4.6)
2. **M1 GRADE-lite + M2 삼각검증** — 출처 강도 정량화·단일출처 제거. (출처 4.2→4.6)
3. **M9 노몰로지컬 네트워크** — 준거타당도 + 데이터 대조점 선점.
4. **M3·M5·M6** — 최신성·저널품질·thesis 보강(중간 비용).
5. **M10 코드북·2차코딩** — 재현성·편향제거(외부 자원 필요).
6. **M4 FACT_CHECK 사람검증** — 사용자 협업.
