# 이음(以音) — 파이프라인별 상세 구현 계획 개요

> **작성 목적**: 재검증보고서([이음_재검증보고서_전체.md](./이음_재검증보고서_전체.md)) 기반, 실제 작동하는 시스템 구현을 위한 파이프라인별 계획 개요.  
> 각 파이프라인은 별도 상세 계획서로 확장 예정.  
> **작성 기준일**: 2026년 6월 1일

## 전체 파이프라인 의존 구조

```
┌─────────────────────────────────────────────────────────────┐
│  Pipeline A — 공공데이터 수집 · 주제 선정                    │
│  (국가기록원 / 민속박물관 / 국악원 API 실연동)               │
└───────────────────────┬─────────────────────────────────────┘
                        │ 주제(미디어 + 질문) 공급
┌───────────────────────▼─────────────────────────────────────┐
│  Pipeline B — 사용자 대화                                    │
│  (노인: STT→AI→TTS / 청년: 텍스트→AI)                       │
└────────────┬──────────────────────────┬──────────────────────┘
             │ 대화 저장                 │ 대화 저장
┌────────────▼───────────┐  ┌───────────▼──────────────────────┐
│  Pipeline C            │  │  Pipeline D — 수필 생성 · 배포   │
│  감정 분석 · 심리 지표  │  │  (topic 필터 수정 + 배포 완성)   │
└────────────┬───────────┘  └──────────────────────────────────┘
             │ 심리 지표
┌────────────▼─────────────────────────────────────────────────┐
│  Pipeline E — 안전 감지 · 긴급 대응                          │
│  (위기 키워드 / 접속 단절 / 감정 악화 → 실제 대응)           │
└────────────┬─────────────────────────────────────────────────┘
             │ 알림 트리거
┌────────────▼─────────────────────────────────────────────────┐
│  Pipeline F — 복지사 알림 · 대시보드                         │
│  (이메일/푸시 실전송 + resolveAlert 수정)                    │
└──────────────────────────────────────────────────────────────┘

  Pipeline G — 데이터 보안 (전 파이프라인 횡단 적용)
  (AES-256 암호화 + 비식별화 + 동의 관리)
```

## Pipeline A — 공공데이터 수집 · 주제 선정

### 현재 문제
- 5개 하드코딩 텍스트 샘플만 존재
- `fetch_from_archives_api()` 정의만 있고 호출 없음
- `media_url` 항상 `None` → 사진/음원 표시 불가
- AI가 주제를 "선정"하지 않음 (단순 산술 인덱싱)

### 구현 목표
```
[공공 API 호출]
국가기록원 / 국립민속박물관 / 국립국악원
        ↓
[미디어 다운로드 · 캐싱]
이미지 URL / 음원 파일 / 설화 텍스트
        ↓
[AI 주제 질문 생성]
Claude API → 연령대별 대화 유도 질문
        ↓
[weekly_topics 테이블 저장]
매주 월요일 자동 스케줄
        ↓
[앱에 미디어 포함 주제 전달]
```

### 핵심 구현 항목
- [ ] 각 공공 API 키 신청 및 연동 (3개 기관)
- [ ] 미디어 타입별(image/audio/story) 파서
- [ ] 주제 질문 AI 자동 생성 로직
- [ ] APScheduler로 매주 월요일 00:00 자동 발행
- [ ] 미디어 파일 S3/로컬 캐시 저장

### 상세 계획서: [pipeline-A/plan_A_public-data.md](./pipeline-A/plan_A_public-data.md)

## Pipeline B — 사용자 대화

### 현재 문제
- STT/TTS 코드는 있으나 `localhost:8000` 하드코딩 → 실기기 동작 불가
- 청년 앱 음악 재생 `media_url=null` → 재생 불가
- 대화 기록에 `topic_id` 연결 안 됨 (항상 `None`)

### 구현 목표
```
[노인]                          [청년]
음성 녹음(expo-av)              텍스트/음악 재생
    ↓ STT                           ↓
Whisper → 텍스트             텍스트 입력
    ↓                               ↓
Claude API (ELDER_SYSTEM)    Claude API (YOUTH_SYSTEM)
    ↓ TTS                           ↓
음성 응답 자동 재생          스트리밍 텍스트 표시
    ↓                               ↓
Conversation 저장            Conversation 저장
(topic_id 포함)              (topic_id 포함)
```

### 핵심 구현 항목
- [ ] API 베이스 URL 환경변수화 (`EXPO_PUBLIC_API_URL`)
- [ ] `Conversation.topic_id` 실제 저장 연결
- [ ] 음악 재생 → media_url 유효 시 expo-av 스트리밍
- [ ] 노인용 TTS 자동 재생 흐름 완성

### 상세 계획서: [pipeline-B/plan_B_user-chat.md](./pipeline-B/plan_B_user-chat.md)

## Pipeline C — 감정 분석 · 심리 지표

### 현재 문제
- `KR-FinBert-SC` 사용 (금융 도메인 특화 → 일상 대화 감정 정확도 의문)
- 감정 1개(긍정/부정/중립)만 구현
- 언어 패턴·참여 패턴·인지 지수 없음

### 구현 목표
```
[대화 텍스트]
    ↓
[감정 분류] KR-FinBert-SC → 긍정/부정/중립
    ↓
[언어 패턴 분석]
· 어휘 다양성 (TTR: Type-Token Ratio)
· 문장 길이 변화 추이
· 반복 표현 비율 (치매 조기 감지)
    ↓
[참여 패턴 분석]
· 일별 접속 빈도
· 대화 길이 변화
    ↓
[심리 지표 종합]
외로움 / 활력 / 인지 / 관계 / 미래 5개 지수
    ↓
[복지사 대시보드 차트]
```

### 핵심 구현 항목
- [ ] 감정 분석 모델 실성능 테스트 (한국어 일상 대화 기준)

## Pipeline D — 수필 생성 · 배포

### 현재 문제
- **Critical Bug**: `essay.py`에서 `topic_id` 필터 없음 → 모든 주제 대화 섞임
- 기여자 수신(push) 미구현
- 프롬프트 v1.0(전문가 설계) 미적용

### 구현 목표
```
[이번 주 topic 확정]
weekly_topics.active_week = 이번 주
    ↓
[해당 주제 대화만 수집]
conversations WHERE topic_id = <this_week_topic_id>
    ↓
[사용자 메시지 ≥ 5건 확인]
    ↓
[프롬프트 버전 선택]
v0 (임시) → v1 (전문가 설계, Phase 2)
    ↓
[Claude API 수필 생성]
max_tokens=1500, 800~1200자
    ↓
[제목 파싱 + DB 저장]
essays 테이블 (topic_id, contributor_cnt)
    ↓
[기여자 전원 배포]
Push Notification (Phase 2) / 앱 수신 화면
```

### 핵심 구현 항목
- [ ] `essay.py` router — `topic_id` WHERE 조건 추가 (즉시 버그 수정)

- [ ] 수필 수신 화면 모바일 연결 (`/essay/latest` 연동)
- [ ] 기여자 목록 추적 (`essay_contributors` 중간 테이블)

## Pipeline E — 안전 감지 · 긴급 대응

### 현재 문제
- 위기 키워드 감지는 되나 텍스트 안내만 (1393 자동 연결 없음)
- 접속 단절: 3일 체크는 되나 복지사에게 실제 알림 미전송
- 감정 악화 추세: 로직 있으나 대시보드와 연결 미흡

### 구현 목표
```
[위기 키워드 감지]
detect_crisis_level() → high/medium/low
    ├─ high → 즉시 1393 딥링크 + 복지사 SMS
    ├─ medium → 1393 텍스트 안내 + 복지사 이메일
    └─ low → DB 기록만
    ↓
[접속 단절 감지]
APScheduler 매일 09:00 실행
check_no_contact() → 3일 이상
    → 복지사 이메일 + 푸시
    ↓
[감정 악화 감지]
APScheduler 매일 실행
check_emotion_trend() → red/yellow
    → 복지사 알림 + 대시보드 🔴🟡
```

### 핵심 구현 항목
- [ ] 모바일 앱 — `Linking.openURL('tel:1393')` 위기 시 자동 팝업
- [ ] APScheduler — 매일 09:00 전체 사용자 안전 점검 job
- [ ] 이메일 발송 (`smtplib` 또는 SendGrid)
- [ ] 심각도별 대응 프로토콜 분기 완성

### 상세 계획서: [pipeline-E/plan_E_safety-detection.md](./pipeline-E/plan_E_safety-detection.md)

## Pipeline F — 복지사 알림 · 대시보드

### 현재 문제
- `resolveAlert()` 더미 (PATCH 요청 미전송)
- 알림 채널 없음 (콘솔 로그만)
- 대시보드 위험도 분류가 단순 (최신 감정만 보고 판단)

### 구현 목표
```
[Pipeline E 알림 트리거]
    ↓
[알림 채널 발송]
이메일(SMTP) + 앱 푸시(FCM) 복지사에게 전송
    ↓
[복지사 대시보드]
· 위험도 분류 고도화 (복합 조건)
· resolveAlert → PATCH /safety/alerts/{id}/resolve 실제 호출
· 개입 이력 기록 및 조회
· 담당 사용자 상세 화면
```

## Pipeline G — 데이터 보안 (횡단 관심사)

### 현재 문제
- 대화 내용 평문 DB 저장
- 비식별화: 익명 닉네임만, 대화 내용은 원문
- 동의서 UI 없음

### 구현 목표
```
[가입 시]
개인정보 수집·이용 동의 UI → DB 동의 이력 저장
    ↓
[대화 저장 시]
content → AES-256 암호화 → DB 저장
복호화: 서비스 내부에서만
    ↓
[분석/보고 시]
비식별화 처리 (user_id → 랜덤 코드)
지역별 집계만 공개
```

### 핵심 구현 항목
- [ ] `cryptography` 라이브러리 — AES-256-GCM 암복호화 유틸
- [ ] `Conversation.content` 저장/조회 레이어 암호화
- [ ] 동의서 UI (모바일 가입 화면)
- [ ] 동의 이력 테이블 (`user_consents`)

### 상세 계획서: [pipeline-G/plan_G_security.md](./pipeline-G/plan_G_security.md)
