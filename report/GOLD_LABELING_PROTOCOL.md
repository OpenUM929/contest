# Micro 라벨 골드 구축 프로토콜 (작업 표준서)
### — 다면평가 → 신뢰할 수 있는 평가 데이터셋 만들기

> **이 문서가 시스템의 천장을 결정합니다.** 분류기는 골드(정답 데이터)보다 정확할 수 없습니다. 골드가 틀리면 95%도 가짜, RAG도 오학습, 인사결정도 방어 불가. 그래서 *"얼마나 제대로 micro 라벨로 분류해 평가셋을 만드느냐"*가 전부입니다.
> 연계: 라벨 규칙은 [`CODEBOOK.md`](CODEBOOK.md), 라벨 정의는 `data/micro_labels`/`dataset/ori`의 `definition/when/not_when`.

---

## 0. 제1원칙 — 골드는 평가받는 모델과 *독립*이어야 한다
> AI가 만든 골드를, 같은 AI가 평가받으면 **순환(circularity)** 입니다. *"정답에 가까운가"*가 아니라 *"AI가 자기와 일치하는가"*를 재게 되어 95%가 무의미해집니다.

따라서 골드는 **(AI 1차 라벨) + (인간 전문가 adjudication)** 으로 만들고, 그 신뢰성은 **κ(일치도) 수치로 증명**합니다. "잘 분류했다"는 *믿어달라*가 아니라 *재서 보여준다*입니다.

---

## 1. 전체 파이프라인
```
[1] 다면평가 원문
   └▶[2] 비식별화(PII 제거)
        └▶[3] 라벨링 단위 분할(문장 + 맥락창)
             └▶[4] AI 1차 라벨링(멀티패스, 증거·근거·신뢰도 강제)
                  └▶[5] 다중 라벨러 대조 + κ 측정
                       ├ κ≥0.7 ─▶[7] 인간 표본 감사 ─▶ 골드 확정
                       └ κ<0.7 ─▶[6] 라벨 정의 수정 → [4] 재라벨링(반복)
                                          └▶[8] 강화풀 / 검증 홀드아웃 분할
```

---

## 2. 단계별 표준

### [2] 비식별화 (착수 전 필수)
- 이름·직급·부서·고유 사건 등 식별자 제거/치환. 개인정보 영향평가(PIA) 완료 후에만 진행.
- 원문↔비식별 매핑은 **분리 보관**(접근 통제). 라벨러는 비식별본만 본다.

### [3] 라벨링 단위 — 맥락을 보존하라
- 기본 단위: **문장**. 단 각 문장에 **맥락창(context_window: 앞뒤 1~2문장)** 을 함께 보관.
- **문장 간 의도**(예: "자유롭게 말하라 … 반대자는 따로 질책")는 단문으로 0% → 이런 건 **다문장 span**을 하나의 단위로 라벨링한다.
- `unit` 필드로 `sentence` / `multi_sentence_span` 구분.

### [4] AI 1차 라벨링 — 직관 금지, 감사 가능하게
규칙:
1. 라벨은 각 micro의 `definition / when / not_when` 으로만 결정(느낌 금지). `not_when`은 **하드 제외**.
2. 각 라벨마다 반드시 출력: **근거 구간(evidence_span) + 사유(reason) + 신뢰도(confidence)**.
3. **멀티라벨 허용**(한 문장 2개 가능), **기권 허용**(애매·저신뢰 → `abstain`, 강제 금지).
4. **멀티패스 self-consistency**: 같은 입력을 N회 라벨 → 일관된 것만 채택, 불일치는 검토 플래그.
5. 과거 회상 서술(~했었다)·역접("하지만 실제로는")·부정형태소("~하지 못했다")는 극성 재검토 트리거.

### [5] 다중 라벨러 + κ — 골드의 신뢰성 증명
- 최소 **2개 독립 라벨링**: 예) AI 멀티패스 vs 인간 전문가(또는 독립 코더).
- **Cohen's κ** 산출(라벨별·전체). 목표 **κ ≥ 0.7**(상당 일치).
- **κ가 낮으면 라벨을 강요하지 말고 *정의를 고친다.*** 불일치 = 보통 "라벨 정의가 애매하다"는 신호. (정의 수정은 `validate_taxonomy`/`CODEBOOK` 절차로.)

### [6] 정의 수정 루프
- 불일치 상위 라벨쌍(혼동표)을 뽑아 `when/not_when` 경계 재정의 → 재라벨링. κ 통과까지 반복.

### [7] 인간 adjudication = 진짜 골드
- AI·라벨러 불일치 건 + 저신뢰 건만 **인간 전문가가 최종 판정**.
- **골드 = adjudication 거친 라벨**. raw AI 출력은 골드가 아니다.

### [8] 강화풀 / 검증 홀드아웃 분할 (누수 방지)
- 확정 골드를 **(a) 강화풀**(벡터DB 적재·RAG exemplar) + **(b) 검증 홀드아웃**(벡터DB에 *안 넣음*, 측정 전용)으로 분할.
- 95%는 **(b)에서만** 측정. (a)로 측정하면 자기참조 → 가짜.
- 같은 피평가자·같은 사건이 양쪽에 겹치지 않게(데이터 누수 차단).

## 3. 골드 항목 스키마 (산출물 JSON)
```json
{
  "id": "G-00001",
  "source": "360_review",
  "unit": "sentence",
  "text": "방법은 자유롭게 정하세요.",
  "context_window": "이번 프로젝트는 자네가 맡아. 방법은 자유롭게 정하세요. 결과만 공유해줘.",
  "ai_labels": [
    {"label_id": "M49-02", "evidence_span": "방법은 자유롭게", "reason": "방식 자율 보장", "confidence": 0.88}
  ],
  "human_labels": ["M49-02"],
  "gold": ["M49-02"],
  "difficulty": "easy",
  "agreement_kappa": 0.81,
  "status": "adjudicated",
  "split": "holdout_test"
}
```
- `gold`: 최종 라벨 배열. `[]` = 해당 라벨 없음(정상), `null` = 보류(미해결).
- `difficulty`: `easy | mirror_flip | reversal | negation | past_tense | cross_sentence | hub`.
- `status`: `adjudicated | pending_review | abstain`.
- `split`: `train_pool | holdout_test`.

## 4. 품질 게이트 (KPI)
| 지표 | 기준 | 왜 |
|---|---|---|
| 라벨러간 일치도 **κ** | ≥ 0.7 | 골드 신뢰성의 증명 |
| **난이도별** 정확도 보고 | 필수 | "전체 95%"가 아니라 *맥락·리스크에서 몇 %* |
| 부정 리스크 라벨 정밀도 | 별도 상향 기준 | 인사 불이익 직결 → 오탐 최소화 |
| 기권율(abstain) | 모니터 | 너무 높으면 정의 불명확, 너무 낮으면 강요 |
| 누수 점검 | 0건 | 강화풀↔홀드아웃 중복 금지 |

> 보고는 **층화(stratified)** 로: easy/mirror_flip/reversal/negation/cross_sentence별 정확도를 따로. 처장·이사회가 보는 건 *어려운 케이스의 수치*다.

## 5. 실행 체크리스트
- [ ] PIA·비식별화 완료, 원문↔비식별 매핑 분리 보관
- [ ] 라벨링 단위(문장+맥락창, cross-sentence는 span) 확정
- [ ] AI 라벨 출력에 evidence_span+reason+confidence 포함
- [ ] 멀티라벨·기권 허용
- [ ] 2개 이상 독립 라벨링 + κ 측정(목표 ≥0.7)
- [ ] κ 미달 라벨 → 정의 수정 → 재라벨링
- [ ] 인간 adjudication으로 골드 확정
- [ ] 난이도 태깅 완료
- [ ] 강화풀/홀드아웃 분할 + 누수 0건 확인
- [ ] 층화 정확도 + 부정 리스크 정밀도 보고

---

## 6. 한 줄 원칙
> **"내가 잘 분류한다"는 주장이 아니라 κ로 증명한다. 골드는 모델과 독립이고, 95%는 홀드아웃에서만 잰다.**
> 이 표준을 지킨 평가셋만 인사평가에 쓸 수 있다.
