**프로젝트명**: 이음(以音) - 은둔 청년 + 고독사 위험 노인 연결 AI 사회안전망 플랫폺

**대상 모듈**: 공공데이터 수집 및 주제 발행 (Dashboard / TopicManager)

---

## 1. 현상 요약

복지사 대시보드 및 주제 관리 페이지에서 **공공 API를 통해 검색한 사진 주제가 출력되지 않음**.  
이미지 후보 카드 및 노인용/청년용 미리보기 화면에서 사진이 표시되지 않고, 빈 영역 또는 플레이스홀더(📷)만 노출됨.

---

## 2. 원인 분석

### 2.1 공공 API 호출 실패 (최상위 원인)

| API | 엔드포인트 | 상태 | 응답 메시지 |
|-----|-----------|------|------------|
| 국립민속박물관 | `api.kcisa.kr/openapi/service/rest/meta/NFMphot` | **403 Forbidden** | `API Key is not valid or is expired / revoked.` |
| 국가기록원 | `apis.data.go.kr/1741050/openapi` | **500 Internal Server Error** | `Unexpected errors` |

- `.env`에 설정된 `PUBLIC_DATA_API_KEY`가 **유효하지 않거나 만료/취소**된 상태임.
- API 서버가 키를 거부하여 이미지 URL을 전혀 가져오지 못함.

### 2.2 이미지 URL 추출 불가

백엔드 코드(`public_data.py`)의 URL 추출 로직:

- **민속박물관**: `identifier` 태그 값이 `http`로 시작하면 `media_url`로 사용
- **국가기록원**: `thumbnailUrl` 또는 `imageUrl` 태그를 `media_url`로 사용

그러나 API가 **200 OK를 반환하지 않으므로**, XML 파싱 단계 자체가 실행되지 않음. 결과적으로 `media_url = None`.

### 2.3 샘플 폴백 데이터에도 이미지 URL 없음

API 실패 시 `TOPIC_POOL` 샘플 데이터로 대체하도록 설계되어 있으나, 모든 샘플의 `media_url`이 **`None` / `null`** 로 설정되어 있음.

```python
# public_data.py TOPIC_POOL 샘플
{
    "title": "1960년대 서울 남대문 시장",
    "media_type": "image",
    "media_url": None,  # ← 이미지 URL 없음
    ...
}
```

→ **API 실패 → 폴백 → media_url 없음 → 이미지 출력 불가** 의 연쇄적 문제 발생.

### 2.4 미디어 캐싱 로직 동작 불가

```python
async def cache_media(url: str, media_type: str) -> str:
    ...
```

- `cache_media()`는 외부 이미지 URL을 다운로드하여 `local_media_cache/`에 저장 후 `/media/{파일명}` 경로를 반환.
- 그러나 `url`이 `None`이거나 API 실패로 인해 유효한 URL이 없으므로, **캐싱 함수 자체가 호출되지 않거나 예외 발생**.
- 실제 `local_media_cache/` 디렉토리 확인 결과 **빈 폴더** 상태.

### 2.5 프론트엔드 경로 매핑 가능성

API가 정상 동작하여 `/media/abc123.jpg` 경로를 반환하더라도, 프론트엔드 개발 서버(예: `http://localhost:5173`)와 백엔드 서버(`http://localhost:8000`)가 **다른 포트**에서 실행 중일 경우, 브라우저가 `/media/...`를 프론트 서버로 요청하여 **404**가 발생할 수 있음.

- 현재 백엔드: `app.mount("/media", StaticFiles(...))`로 `/media/` 경로 서빙 중
- 프론트엔드: `<img src="/media/abc123.jpg" />` → 브라우저가 프론트 포트로 요청

→ **CORS/경로 불일치 문제 가능성 존재** (현재는 API 실패로 인해 재현되지 않음)

## 5. 해결 방안

### 5.1 단기 방안 (API 키 복구 전까지)

1. **샘플 데이터(`TOPIC_POOL`)에 실제 이미지 URL 삽입**
   - Unsplash, 위키미디어 등 퍼블릭 도메인 이미지 URL을 샘플에 추가
   - `Dashboard.tsx` 폴백 샘플에도 동일하게 적용
   - API 실패 시에도 사용자에게 이미지가 보이도록 함

2. **`<img>` 태그 에러 핸들러 추가**
   - 이미지 로드 실패 시 `onError`로 플레이스홀더(📷)가 표시되도록 개선

3. **프론트엔드 `media_url` 상대 경로 처리**
   - `/media/...`로 시작하면 `http://localhost:8000`을 자동으로 붙이도록 수정

### 5.2 중기 방안 (API 복구 후)

1. **공공데이터포털 API 키 재발급/활성화**
   - `www.data.go.kr` 접속 → 마이페이지 → API 키 상태 확인
   - 필요 시 재발급 후 `.env`의 `PUBLIC_DATA_API_KEY` 업데이트

2. **API 응답 필드명 재확인**
   - 실제 XML 응답에서 이미지 관련 태그명이 `identifier`, `thumbnailUrl`, `imageUrl` 등과 일치하는지 확인
   - 공공 API는 필드명이 변경되는 경우가 빈번함

3. **이미지 캐싱 전략 개선**
   - `cache_media()` 실패 시 원본 외부 URL을 그대로 `media_url`로 반환하는 폴백 추가
   - 캐시 디렉토리에 파일이 없으면 재다운로드 시도

### 5.3 장기 방안

1. **Unsplash / 위키미디어 Commons 등 대체 이미지 소스 연동**
   - 공공 API 장애 시 자동으로 대체 소스에서 이미지 검색
   - 서비스 가용성 향상

2. **통합 모니터링**
   - API 응답 상태, 캐시 히트율, 이미지 로드 성공률을 로깅/알림

---

## 6. 결론

현재 사진 출력 문제의 **근본 원인은 공공 API 키 만료/취소**입니다.  
API가 응답하지 않으므로 이미지 URL을 전혀 가져올 수 없고, 이로 인해 프론트엔드 어디에서도 실제 이미지를 표시할 수 없습니다.

**즉시 조치**: 샘플 데이터에 실제 이미지 URL을 넣어 API 복구 전까지 정상 동작하게끔 수정하는 것을 권장합니다.  
**필수 조치**: 공공데이터포털에서 API 키를 재발급/활성화하여 실제 공공데이터 연동을 복구해야 합니다.
