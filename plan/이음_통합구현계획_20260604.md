## 1. 전체 파이프라인 의존 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│  Pipeline A/A'/A'' — 공공데이터·주제선정·설문지통합관리               │
│  (공공 API / 복지사 듀얼 협의 / choice·narrative·mixed 질문 생성)      │
└────────────────────────────┬────────────────────────────────────────┘
                             │ 주제(미디어 + 질문) 공급
┌────────────────────────────▼────────────────────────────────────────┐
│  Pipeline B — 사용자 대화                                              │
│  (노인: STT→AI→TTS / 청년: 텍스트→AI / 동적 설문지 응답)              │
└────────────┬──────────────────────────────┬─────────────────────────┘
             │ 대화 저장                      │ 설문지 응답 저장
┌────────────▼─────────────┐  ┌──────────────▼─────────────────────────┐
│  Pipeline C              │  │  Pipeline D — 수필 생성 · 배포         │
│  감정 분석 · 심리 지표    │  │  (topic_id 필터링 + 기여자 추적)       │
└────────────┬─────────────┘  └────────────────────────────────────────┘
             │ 심리 지표
┌────────────▼─────────────────────────────────────────────────────────┐
│  Pipeline E — 안전 감지 · 긴급 대응                                   │
│  (위기 키워드 / 접속 단절 / 감정 악화 → 실제 대응)                     │
└────────────┬─────────────────────────────────────────────────────────┘
             │ 알림 트리거
┌────────────▼─────────────────────────────────────────────────────────┐
│  Pipeline F — 복지사 알림 · 대시보드                                  │
│  (이메일/푸시 실전송 + resolveAlert + 개입 이력)                       │
└──────────────────────────────────────────────────────────────────────┘

  Pipeline G — 데이터 보안 (전 파이프라인 횡단 적용)
  (AES-256 암호화 + 비식별화 + 동의 관리)
```
